## 🧪 Класс Library

Реализует простую библиотеку книг. Методы:

| Метод | Описание |
|-------|----------|
| `add_book(title, author)` | Добавляет книгу; если уже есть – `ValueError`. |
| `issue_book(title)`       | Выдаёт книгу; если нет или уже выдана – `ValueError`. |
| `return_book(title)`      | Возвращает книгу; если нет или не выдана – `ValueError`. |
| `get_available_count()`   | Возвращает количество доступных книг. |
| `get_issued_count()`      | Возвращает количество выданных книг. |
| `get_book_info(title)`    | Возвращает словарь с информацией о книге или `None`. |

## 🔧 Фикстуры (conftest.py)

### 1. `library` (yield)
- Создаёт библиотеку с тремя книгами.
- Перед `yield` выводит `"Setup: библиотека готова"`.
- После `yield` выводит `"Teardown: библиотека очищена"`.
- Используется в тестах, где нужно менять состояние.

### 2. `empty_library` (scope="function")
- Возвращает пустую библиотеку.
- Каждый тест получает свой экземпляр.

### 3. `library_readonly` (scope="module")
- Создаёт библиотеку с пятью книгами (все доступны).
- Один экземпляр на весь модуль (id одинаковый).

### 4. `log_tests` (autouse=True)
- Автоматически выводит разделитель перед и после каждого теста с названием теста.

## 📝 Тесты (test_library.py)

### Задание 1 – Параметризация
- `test_add_book_valid` – 4 корректных набора данных для `add_book`.
- `test_add_book_invalid` – 4 невалидных набора (ожидается `ValueError`).
- `test_get_available_count` – 4 разных состояния библиотеки.

### Задание 2 – Scope
- Тесты на `library_readonly`: два теста, проверяют одинаковость `id` фикстуры.
- Тесты на `empty_library`: два теста, проверяют разность `id` (т.к. scope=function).

### Задание 3 – yield
- Три теста, использующие фикстуру `library`. Каждый тест начинает с трёх доступных книг и проверяет изменение состояния через `issue_book` и `return_book`.

### Задание 4 – autouse
- Фикстура `log_tests` автоматически выводит разделители, видимые при запуске с `-s`.

## 🚀 Запуск

1. **Установка зависимостей** (при необходимости):
   ```bash
   pip install pytest
   ```
   
2. **Запуск тестов** (из корня проекта, где папки src и tests):
   ```bash
   pytest -s -v
   ```
- -s – отключить захват вывода (чтобы видеть сообщения из фикстур).
- -v – подробный вывод.

3. Ожидаемый результат: все тесты зелёные, в выводе видны разделители, сообщения о настройке/очистке фикстуры library, а также одинаковые/разные id в соответствии с scope.
